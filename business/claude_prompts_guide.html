調査・分析

01

複数ソースを一発で統合する

5本の記事を別々に読むのをやめる

5本の記事を全て同じ会話にペーストして、一つの矛盾のないブリーフに蒸留するよう頼む。各記事が見落としたことを拾い、どこで矛盾しているかを発見する。90分かかる読書が3分になる。

📍 おすすめの使用場面

業界調査・競合リサーチ・勉強など複数の記事や資料を読む必要がある時。読む前にまとめてペーストするだけで全体像を把握できる。

プロンプト（日本語訳）
あなたはシニアリサーチアナリストです。複数のソースを、どの個別ソースよりも有益な一つの決定的なブリーフに統合するのがあなたの仕事です。

以下は[テーマ]に関する[数]本の記事・論文・レポートです：

[全ソースをここにペースト]

以下の5つのセクションで構造化された統合を作成してください：

1. 共通コンセンサス — 全ソースが同意していること（3〜5点）
2. 主要な矛盾 — ソース間で矛盾している点とその理由
3. 重大なギャップ — どのソースもカバーしていないが、このテーマを理解するために重要なこと
4. 最強の主張 — 全ソースの中で最も証拠に基づいた洞察
5. 実行可能なアクション — これに基づいて私が行うべき具体的な一つのこと

ルール：
- 各記事を別々に要約しない。全体を横断して統合すること
- 最も信頼できるソースとその理由を示すこと
- 一つのソースにしか登場しない主張は「未確認」とマークすること
- 合計600語以内

📍 おすすめの使用場面

転職・起業・投資など人生の大きな決断をする直前に。自分では気づけない致命的な穴を事前に発見できる。

プロンプト（日本語訳）
あなたは優秀で容赦ない批評家です。このアイデアを完膚なきまでに叩き潰すのが唯一の仕事です。丁寧にではなく、建設的にでもなく。ひびを全て見つけてください。

私のアイデア・計画・決断：[できるだけ詳細に説明]

以下の4つの次元から攻撃してください：

1. 誤った前提 — 間違っているかもしれない思い込みは何か？
2. 実行リスク — 実際にはどこで失敗しそうか？
3. 市場・人間の現実 — 人やシステムの実際の動き方で無視していることは何か？
4. 致命的な欠陥 — これを完全に殺せる一つのことがあるとしたら、それは何か？

ルール：

03

反対意見を最高の形で代弁させる

スティールマン：賢い反対側の論理を理解する

何かに反対している。それを切り捨てる前に、Claudeにスティールマン（最善の解釈）を作らせる。意見を変えるためではなく、反対側の賢い人が実際に何を信じているかを理解するために。

📍 おすすめの使用場面

議論・交渉の前、または自分の考えが偏っていると感じた時。反対意見を持つ賢い人の論理を理解することで視野が広がる。

プロンプト（日本語訳）
私はこの立場に反対しています：[立場を明確に述べる]

私の現在の見方：[なぜ反対なのか]

反対の立場の最強のスティールマンを作ってください。この見解を持つ最も賢く、最も情報を持った人たちが深く考えた上で良い理由があると仮定してください。

以下の構成で：

1. 核心的な洞察 — この立場が根拠としている根本的な真実は何か？
2. 証拠 — これを最も強く支持するデータ・歴史・論理は何か？
3. 私の見方の最も弱い点 — この立場は私自身の思考のどんな盲点を明らかにしているか？
4. 最強の一つの論点 — この立場の優秀な支持者が言いそうな一段落

ルール：
- スティールマンへの反論や留保を含めない。それは私の仕事
- あなた個人がこの立場に反対だと言わない
- 本当にそれを信じているかのように議論すること
- 目標：「賢い人がなぜこれを信じるのかわかった」と私が思えること
04

メンタルモデルで学ぶ

知識ではなく、思考の枠組みを手に入れる

理解できていないトピックを、一連の繋がったメンタルモデルとして教えてもらう。事実でも要約でもない。考える枠組みを。10分後には誰かに説明でき、新しい状況に応用できる。「知っている」と「理解している」の違いがここにある。

📍 おすすめの使用場面

新しい分野（経済・テクノロジー・医療など）を学び始める最初のステップとして。事実の暗記ではなく、考えるための枠組みを手に入れられる。

プロンプト（日本語訳）
あなたは世界クラスの教師です。私に情報を与えるのが仕事ではなく、私が考えられる枠組みを与えることが仕事です。

テーマ：[テーマ]
現在のレベル：[ゼロ / 基礎知識あり / 中級]
これを理解したい理由：[目標またはコンテキスト]

3〜5つのメンタルモデルを使ってこのテーマを教えてください。各モデルについて：

- 名前：このモデルは何と呼ばれているか？
- 核心的なアイデア：全く初めての人に2〜3文で説明する
- ここへの応用：このモデルが[テーマ]についての何かをどう説明するか具体的に
- 具体例：すぐにイメージできる実世界の例
- よくある誤解：このモデルを使わないとどんな間違いを犯しやすいか？

全モデルの後：それらが互いにどう繋がるかを示してください。全てを理解することで、どれか一つだけでは見えなかった何が見えてくるか？

最後に：私が正しく理解していれば答えられるはずの一つの問いを提示してください。

文章作成

05

自分の文体を学習させる

AI文ではなく、あなたの声で書かせる

自分の文章サンプルを3本ペーストして、文体を分析させる。リズム、語彙、段落の開き方と締め方。そしてテーマを渡して自分の声で書かせる。空白のページから自分で書くより、あなたらしい文章が出てくる。

📍 おすすめの使用場面

ブログ・SNS・メルマガなど定期的に発信している人に。自分の文体をClaudeに覚えさせれば、ネタだけ渡せば記事が完成する。

プロンプト（日本語訳）
あなたは文体アナリストとゴーストライターです。私の書き方を学んでください。何を言うかだけでなく、どう言うかを。そしてそれを完全に再現してください。

私の文章サンプル3本：

サンプル1：[ペースト]
サンプル2：[ペースト]
サンプル3：[ペースト]

ステップ1 — 以下の次元で文体を分析してください：
- 文の長さとリズム（短くてパンチがある / 長く流れる / 混在）
- 語彙レベル（シンプル / 専門的 / 会話的）
- 段落の開き方
- アイデアの締め方
- 私が絶対にやらないこと（受動態 / 埋め言葉 / 特定の構造）
- 「シグネチャームーブ」— 私の文章を瞬時に認識させる一つのこと

ステップ2 — 私の声で[形式：スレッド / 記事 / メール / 投稿]を[テーマ]について書いてください。

ステップ2のルール：
- 声のスタイルを私が編集せずに発表できるほど忠実に合わせること
- 私が持たないようなアイデアは加えない
- 私のスタイルがカジュアルならフォーマルにしない。パンチがあるなら水増ししない
06

容赦ない編集者

下書きを「改善」ではなく「破壊」してもらう

下書きをペーストして「改善して」ではなく「破壊して」と頼む。読者の時間を無駄にする全ての文、根拠のない全てのアイデア、削れる全ての言葉を見つけてもらう。これを生き残ったバージョンだけが公開する価値がある。

📍 おすすめの使用場面

記事・提案書・企画書を公開・提出する直前の最終チェックに。自分では見えない冗長な部分を容赦なく削ってもらえる。

プロンプト（日本語訳）
あなたは私が今まで出会った中で最も厳しい編集者です。励ましは不要。機能していること・機能していないこと・削除すべきことの正直な評価だけをください。

私の下書き：[下書きをペースト]

5つの次元で編集してください：

1. カット — 読者の時間を無駄にしている全ての文や段落をリストアップ。引用して、なぜ何も追加していないかを説明
2. 弱いアイデア — どの主張が曖昧で、根拠がなく、その場にいる価値がないか？
3. 欠けているもの — ここにあるべきで存在しないものは何か？読者が持っているが私が答えていない質問は？
4. 構造 — 順序は正しいか？移動すべきものがあるか？冒頭は注意を引いているか？結末は決まっているか？
5. 最大の一つの問題 — もし一つだけ直せるとしたら、それは何か？

そして：編集済みのバージョンを作成してください。容赦なく切る。全ての文を引き締める。削った部分の代わりに水増ししない。

ルール：
- 「Xは良い点だが…」と批判前に言わない。ただ編集する
- 下書きが根本的に壊れているなら、直接そう言う
- 目標は読者が最後まで読み切るバージョン。私のエゴを守るバージョンではない
07

一つのテキストを5つの読者向けに書き換える

同じアイデアが全く違う言葉になる

一人向けに何かを書いた。5つの異なる読者向けに書き換えてもらう。核心的なアイデアは同じ。5歳児、専門家、懐疑的な人、ジャーナリスト、CEO。これを通じて自分のアイデアについてどんな演習よりも多くのことを学べる。

📍 おすすめの使用場面

プレゼン・説明資料・マニュアル作成時に。「誰に向けて書けばいいかわからない」という悩みを、5パターンの比較で一気に解消できる。

プロンプト（日本語訳）
あなたはマスターコミュニケーターです。一つのアイデアを取り、全く異なる読者向けに完璧に翻訳したときにどう聞こえるかを示してください。

私の元のテキスト：[テキストをペースト]

5つの読者向けに書き直してください。各バージョンでその読者に完全にコミットしてください。

バージョン1 — 完全な初心者：
専門用語なし。類推を使う。何も知らないが知性はあると仮定。

バージョン2 — 分野の専門家：
基礎はスキップ。正確な用語を使う。元より深く掘り下げる。

バージョン3 — 懐疑的な人：
これが重要だと信じていない。最強の証拠から始める。反論を先読みして先に答える。

バージョン4 — ジャーナリスト（見出し＋2文）：
一つのフック見出し。最もニュース価値のある角度を捉える2文。クリックさせる文章。

バージョン5 — 経営幹部（30秒）：
最大3文。それが何か、なぜ重要か、何をすべきか。文脈なし、前置きなし。

5バージョンの後：どのバージョンが最も書くのが難しく、その理由は何かを教えてください。
08

箇条書きから記事へ

メモの断片を完成した文章に変換する

大雑把なメモ、箇条書き、断片がある。構造化された完全な作品に変換してもらう。目標の長さ、対象読者、トーンを指定する。箇条書きが骨格になり、Claudeが肉付けをする。あなたはフォーマットではなく考えることにエネルギーを使う。

📍 おすすめの使用場面

アイデアはあるが文章化が苦手・時間がない時に。箇条書きのメモを渡すだけで、構成・つなぎ・結論まで整えた完成品が出てくる。

プロンプト（日本語訳）
あなたはプロのライターと編集者です。私には大雑把なメモがあります。私のアイデアを失わず、私が持っていないアイデアを追加せずに、完成した作品に変えてください。

テーマ：[テーマ]
形式：[記事 / 投稿 / レポート / メール]
目標文字数：[文字数]
読者：[誰が読み、何を気にしているか]
トーン：[フォーマル / 会話的 / 直接的 / 専門的]
目標：[読んだ後に何を考え、感じ、すべきか]

私の大雑把なメモ：[箇条書き / 断片 / 大雑把なアイデアをペースト]

指示：
- リストした全てのアイデアを使う。何も落とさない
- アイデア間のつなぎと文脈を埋める — それがあなたの仕事
- 私が含めなかった新しいアイデアを追加しない。私のアイデアは広げて良いが、新しいものは発明しない
- 注意を引く強い冒頭と、決まる締めくくりで構成する
- メモが不明確または矛盾している場合は、推測せず最後にフラグを立てる

完成した作品を届け、その後、使い方がわからなかった断片があればメモを一行つけてください。

キャリア

09

給与交渉のロールプレイ

本番前に何度も練習して自信をつける

ほとんどの人は一度も練習したことがないから給与交渉で何千ドルも損をする。Claudeに未来の雇用主を演じさせ、コンテキスト・オファー・想定反論を与える。リアルタイムで交渉する。本番前に12回練習できる。

📍 おすすめの使用場面

転職・昇給交渉の1〜2週間前から毎日練習。実際の場面を想定したロールプレイを繰り返すことで、本番で言葉に詰まらなくなる。

プロンプト（日本語訳）
あなたは[会社タイプ：スタートアップ / 大企業 / エージェンシー]の採用マネージャーです。経験豊富でプロフェッショナルで、簡単には折れません。予算の制約とチームダイナミクスを守る必要があります。

役職：[職種]
テーブルに出ているオファー：[現在のオファー]
私の実際の目標：[欲しいもの]
私の最強のレバレッジ：[競合オファー / 特定スキル / 独自の経験]
私の最も弱い点：[経験のギャップ / キャリアチェンジ / 競合オファーなし]

私とリアルな交渉ロールプレイをしてください。オファーを確認するところからあなたが始めます。

採用マネージャーとしてのルール：
- 交渉中に以下の反論を少なくとも3つ使う：「このレベルは予算が固定」「チームに不公平」「市場データではこの役職はX」「6ヶ月後に見直しましょう」
- 簡単に折れない。全てのドルを勝ち取らせる
- 私が弱い論点を出したら、具体的な反論で押し返す
- 私が強い論点を出したら認めるが、それでも交渉する

ロールプレイ後：フィードバックをください。うまくできたことは？テーブルに残した論点は何か？最も効果的だったムーブは何か？

面接シミュレーター

JDを貼るだけで本番さながらの練習ができる

求人票をペーストして面接官として演じてもらう。その役職が出しやすい質問をそのまま聞いてくる。答えると採点される：何が刺さったか、何が曖昧だったか、代わりに何を言うべきだったか。3セッションやれば、どんな面接でも落ち着いて臨める。

📍 おすすめの使用場面

面接本番の1週間前から毎日練習。求人票をペーストするだけでその会社向けの質問が出てくるので、汎用的な練習より圧倒的に実践的。

プロンプト（日本語訳）
あなたはリアルな面接を行うシニア面接官です。プロフェッショナルで注意深く、簡単には通しません。

求人票：[求人票全文をペースト]

私のバックグラウンド：[あなたの経験と強みについて2〜3文]
面接の種類：[行動面接 / 技術面接 / ケース面接 / 混合]

面接を実施してください。一度に一問ずつ。私の答えを待ってから次に進む。

各回答の後に構造化されたフィードバックを：
- 強み：答えで機能したこと
- 弱み：曖昧・欠けている・説得力のなかったこと
- こう言うべきだった：私が使わなかったキーポイントや構造
次の質問に進んでください。

以下の分野をカバーしてください：役職への動機・STARフォーマットを使った関連する過去の経験・特定の課題への対応・働き方とチームワーク・求人票に基づく技術/専門的な質問・一つの変化球質問

最後の質問の後：総合評価を。採用判断（はい/いいえ/保留）とその判断に最も影響した一つのことを。
11

職務経歴書の掛け算

1つの経歴書を3つの求人向けに最適化

一つの経歴書がある。でも応募する求人は全て違う。経歴書と3つの異なる求人票をペーストして3バージョンを作ってもらう。それぞれがその役職にとって最も重要な経験を前面に出す。同じ事実。全く違うエンファシス。

📍 おすすめの使用場面

複数の会社・職種に同時応募する転職活動中に。一つの経歴書で全部カバーしようとする非効率をやめられる。

プロンプト（日本語訳）
あなたはプロの経歴書ライターとキャリア戦略家です。一人の実際の経験を取り、嘘をついたり誇張したり重要な省略をしたりせずに、異なる役職向けに異なる形で位置付けるのが仕事です。

私の現在の経歴書：[経歴書全文をペースト]

ターゲットの求人票3つ：

求人1：[求人票をペースト]
求人2：[求人票をペースト]
求人3：[求人票をペースト]

各求人について以下の変更を加えたカスタマイズ版を作成：
- その役職にとって最も関連性の高い経験が先に来るよう箇条書きを並べ替え
- その求人票の正確な言葉とキーワードを使って箇条書きを書き直す
- その会社が求めていることに直接語りかけるようサマリー/ヘッドラインを調整
- その特定の役職に無関係な経験は削除または軽視する

ルール：
- 全ての事実は正確に。何も発明・誇張しない
- 強い動詞を使い、元に数字があれば結果を数値化する
- 求人が必要とするものと私の経歴書が示すもとの間の重大なギャップをフラグを立てる

3バージョンの後：最も強いフィット感を持つ応募はどれか、最大のギャップがあるのはどれかを教えてください。
12

難しい会話の事前準備

上司・クライアントとの対話を事前にシミュレート

難しい会話がある。上司、クライアント、共同創業者との。ほとんどの人は準備不足で臨み、言い過ぎるか言い足りないかになる。状況を描写して会話をマッピングしてもらう：何を言うか、相手が何と言い返すか、各シナリオへの対応方法を。

📍 おすすめの使用場面

上司への昇給・異動の相談、クライアントへのクレーム対応、パートナーとの重要な話し合いの前夜に。即興ではなく「準備した会話」に変わる。

プロンプト（日本語訳）
あなたはエグゼクティブコーチとコミュニケーション戦略家です。難しい会話が迫っています。あらゆるシナリオに備えて準備を整えてください。

話す相手：[役職と私との関係]
会話のテーマ：[トピック/問題]
何が起きたか：[これに至った全コンテキスト]
私の目標：[最後に達成したいこと]
私の恐れ：[彼らが言うか行うと最も心配していること]
絶対にしてはいけないこと：[レッドライン — 謝らない / 引かない / など]

以下を教えてください：

1. オープニングライン — 正確な始め方。明確で直接的で攻撃的でない
2. 彼らの最も可能性の高い3つの反応 — コンテキストに基づいて、おそらく何を言うか
3. 各反応への私の答え — 各シナリオへの対処の言葉
4. 避けるべき罠 — その瞬間に私が最もやりそうで会話を脱線させることは何か
5. 終わり方 — 会話がどんな展開になっても、明確に締める方法

ルール：
- 相手を簡単にしたり合理的にしすぎない。押し返しに備えさせる
- 企業言語を使わない。実際に言える言葉を
- コンテキストを考えると私の目標が非現実的なら、入る前に言ってほしい

法律文書の翻訳機

サインする前に本当に何に同意するかを知る

契約書、利用規約の更新、賃貸契約、プライバシーポリシーを受け取った。内容の半分が理解できない。ペーストして平易な日本語に翻訳してもらう。何に同意しているか、危険な条項はどれか、サインする前に聞くべきことは何か。

📍 おすすめの使用場面

契約書・雇用契約・NDA・利用規約など重要書類を受け取ったその日に。弁護士に相談する前の「自分で理解する」ステップとして必ず通しておく。

プロンプト（日本語訳）
私はサインする前にこの文書を理解する必要があります。あなたは法的アドバイスを提供しているのではなく、この文書が実際に何を言っているかを平易な言葉で私が理解できるよう助けています。

文書タイプ：[契約書 / 賃貸 / 利用規約 / NDA / 雇用契約 / その他]

文書：[全文をペースト]

以下のセクションで分析してください：

1. 私が同意していること — 私側の核心的な義務を平易な言葉で要約。私が行うこと・支払うこと・しないことを約束していること
2. 彼らが約束していること — 私への彼らの義務は？
3. 最も危険な3つの条項 — 何かがうまくいかないときに最も傷つく可能性のある条項は？各条項を引用し、リスクを平易な言葉で説明
4. 異常または標準外 — このタイプの文書では典型的でないものは何か？驚くべきことは？
5. 欠けているもの — このタイプの合意に通常現れる条項で、ここにないものは何か？それは私を守るか晒すか？
6. サインする前に聞くべき質問 — 弁護士または相手方に提起すべき5つの具体的な質問

注意：これは私自身の理解のためのみで、法的アドバイスではありません。

個人財務の分析

「外食を減らせ」ではなく具体的な漏れを見つける

先月の支出をペーストして、カテゴリ分けして、パターンを見つけて、どこでお金が漏れているかを正確に特定してもらう。「外食に使いすぎ」という曖昧なアドバイスではない。「今月ほとんど使っていないサブスクに3万円使っている」という具体性が行動を変える。

📍 おすすめの使用場面

月末の家計振り返りに。「なんとなくお金が貯まらない」という感覚を、具体的な数字と原因に変えることで初めて改善できる。

プロンプト（日本語訳）
あなたは個人財務アナリストです。実際の支出データを見て、具体的で正直な評価を行う仕事です。一般的なアドバイスではなく。

税引き後の月収：[金額]
主な財務目標：[緊急資金の構築 / 借金返済 / 投資 / X のための貯金]
その目標のタイムライン：[月数/年数]
削れない固定費：[家賃 / ローン返済 / など]

先月の支出：[支出リストまたは明細書をペースト — 項目と金額]

以下を分析して作成してください：

1. カテゴリ別内訳 — 全支出を以下にグループ化：住居 / 食費 / 交通 / サブスク / 娯楽 / 健康 / 貯蓄 / その他。カテゴリ合計と収入に占める割合を表示

2. 3つの最大の漏れ — 「外食に多く使っている」ではなく具体的に：どのカテゴリ、いくら、私の収入レベルの合理的なベンチマークと比較して

3. 即効性のある改善 — 生活への影響を最小限に今すぐ削減できるものは？各項目の正確な月間節約額を計算

4. 目標の実現可能性 — 収入と支出を考えると、述べたタイムラインで目標を達成できるか？できないなら何を変える必要があるか？

5. 一つのこと — 今月最大の財務上の差を生む行動変容を一つ選ぶとしたら？

具体的に。実際のデータから数字を引用すること。どんなブログにも書いてあるような一般的なアドバイスは不要。

ファインマン式チューター

理解を確認しながら段階的に教えてもらう

ファインマンテクニック：シンプルに説明できるとき、本当に理解している。Claudeにこの方法で教えてもらう。シンプルに始め、理解できたか確認して、準備できたときだけ複雑にしていく。コンセプトが固まるまで先に進まない。読書ではなく学習だ。

📍 おすすめの使用場面

資格勉強・新しい技術の習得・「わかったつもり」になっている概念の再確認に。理解できたかを確認しながら進むので、知識が定着しやすい。

プロンプト（日本語訳）
あなたはファインマンテクニックを使うマスター教師です。ルールは一つ：学生が前のコンセプトを本当に理解するまで次に進まない。ただうなずくのではなく、本当に理解するまで。

理解したいテーマ：[テーマ]
現在のレベル：[ゼロ / 基礎知識あり / 中級]
これを理解したい理由：[実際のユースケースまたは目標]
最も学びやすい方法：[類推 / 例 / 視覚的な説明 / ステップごとのロジック]

セッションの進め方：

1. 最も基本的なコンセプトから始める。最もシンプルな言葉で説明する。専門用語なし
2. 説明の後、私が理解したかテストする一つの質問をする — 暗記問題ではなく理解問題
3. 私の答えを待つ
4. 正解なら：簡潔に認めて次の複雑さの層へ進む
5. 間違いまたは部分的な正解なら：同じ説明を繰り返さない。別の類推や枠組みを見つける。再試行
6. 一度に一つのコンセプト以上を説明しない
7. 日常生活からの類推を使う — その分野自体からではなく

私がテーマの全体像を理解するまで続ける。

最後に：テーマを自分の言葉で私に説明させてください。正しかったこと、抜けていたこと、間違っていたことを教えてください。

20

ソクラテスモード

答えを教えてもらうのではなく、気づかせてもらう

これがClaudeで最も使われていないもの。答えを求める代わりに、質問で教えてもらう。テーマを伝えると、質問してくる。答えると、その答えを基に積み上げていき、ギャップを修正し、自分の考えで理解へと導いてくれる。受動的な読書は親しみを生む。ソクラテス式の対話は知識を生む。その違いはすぐにわかる。

📍 おすすめの使用場面

コーチングを受けたい時・思考の癖に気づきたい時・自己理解を深めたい時。答えを与えてもらうのではなく、質問を通じて自分で気づく力を鍛えられる。

プロンプト（日本語訳）
あなたはソクラテス式教師です。説明はしません。質問をします。その質問を通じて、私が自分で答えを発見するよう導いてください。これは最も要求が高い形の教育であり、最も効果的です。

テーマ：[テーマ]
目標：[理解したいこと、またはできるようになりたいこと]
現在のレベル：[今知っていると思っていること / 完全初心者]

必ず守るルール：

1. まずこのテーマについて私が今信じていることを明らかにする一つの質問から始める
2. 直接答えを教えない。フォローアップの質問で導く
3. 私が間違っている場合、訂正しない。私が間違っていると気づかせる質問をする
4. 私が正しい場合、さらに深く掘り下げる。次の質問を聞く
5. 「説明してください」と求めても拒否する。「答えは何だと思いますか？」と言って続ける
6. 各質問は一文。前置きなし
7. 一度に二つ質問しない

私が主要な洞察を自分の言葉で表現できたときのみセッションを終了する。

最後に：一度だけキャラクターを外れて — 私を導こうとしていた重要な洞察は何か？対話のどの瞬間に理解したとわかったか？もし続けていたら次の質問は何だったか？
