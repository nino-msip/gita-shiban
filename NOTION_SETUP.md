# Notion ワークスペース設計書
## MoritaSaki Entertainment

> このドキュメント通りに作れば、経営管理の全てがNotionで完結します。
> 所要時間：初期設定 約60〜90分

---

## ワークスペース全体構造

```
MoritaSaki Entertainment（ワークスペース）
│
├── 🏠 ホーム                    ← 毎朝ここを開く。全体の状況一覧
├── ✅ タスクDB                  ← 週間・月間・緊急タスクの管理
├── 📅 コンテンツカレンダーDB    ← 投稿草稿・スケジュール管理
├── 📊 KPI・チケット管理         ← 数値の定点観測
├── 📁 アーカイブ
│   ├── 投稿済み草稿
│   └── 完了タスク
└── 📎 リソース
    ├── バンド基本情報
    ├── メディアリスト（ピッチ先）
    └── チケットリンク集
```

---

## ステップ1：アカウント作成

1. https://notion.so にアクセス
2. 「無料で始める」→ Googleアカウントでログイン（推奨）
3. ワークスペース名を **「MoritaSaki Entertainment」** に設定

> 無料プランで今回の用途は全て賄えます。

---

## ステップ2：タスクDB の作成（最重要）

### 作り方
1. 左サイドバー「＋ ページを追加」
2. タイトルに **「✅ タスクDB」** と入力
3. 「テーブル」を選択（データベース）

### 追加するプロパティ（列）

| プロパティ名 | 種類 | 設定内容 |
|------------|------|---------|
| タスク名 | タイトル | （デフォルト） |
| ステータス | ステータス | 未着手 / 進行中 / 完了 |
| 優先度 | セレクト | 🔴 緊急 / 🟡 今週 / 🟢 月内 |
| カテゴリ | セレクト | SNS / ライブ / ビジネス / プレス / 物販 |
| 期日 | 日付 | — |
| 担当 | テキスト | （自分の名前 or メンバー名） |
| メモ | テキスト | 補足メモ |

### ビュー（表示方法）を3つ作る

**① テーブルビュー「全タスク」**（デフォルト）
- フィルター: なし
- ソート: 優先度 → 期日

**② ボードビュー「ステータス別」**
1. 「＋ ビューを追加」→「ボード」
2. グループ化: ステータス
3. 名前: **「📋 今週のボード」**

**③ カレンダービュー「期日カレンダー」**
1. 「＋ ビューを追加」→「カレンダー」
2. 表示日付: 期日
3. 名前: **「🗓 締切カレンダー」**

### 初期タスクの登録
以下をそのままコピーして登録してください：

| タスク名 | 優先度 | カテゴリ | 期日 |
|---------|-------|---------|------|
| Bufferアカウント作成・XとInstagramを接続 | 🔴 緊急 | SNS | 今週 |
| 大阪チケット会社へ状況確認の連絡 | 🔴 緊急 | ライブ | 今週 |
| 全SNSプロフィールのリンクをチケットURLに変更 | 🔴 緊急 | SNS | 今週 |
| InstagramをクリエイターアカウントへSW | 🟡 今週 | SNS | 今週 |
| posts/2026-05-week3.md の草稿を確認・承認 | 🟡 今週 | SNS | 5/19 |
| 草稿の（チケットURL）箇所にURLを入力 | 🟡 今週 | SNS | 5/19 |
| 英語プレスキット（バイオ+写真）の作成 | 🟢 月内 | プレス | 5/31 |
| Bandcamp Daily へのピッチ（英語メール） | 🟢 月内 | プレス | 5/31 |
| Time Out Tokyo にライブ情報を掲載申請 | 🟢 月内 | ライブ | 6/7 |
| Spotify for Artists — 週次数値を確認 | 🟢 月内 | SNS | 毎週月曜 |

## ステップ3：コンテンツカレンダーDB の作成

### 作り方
1. 「＋ ページを追加」→ タイトル **「📅 コンテンツカレンダーDB」**
2. 「テーブル」を選択

### プロパティ

| プロパティ名 | 種類 | 設定内容 |
|------------|------|---------|
| 投稿タイトル | タイトル | 管理用の名前（例：5/21 Reels①） |
| 投稿本文 | テキスト | 実際に投稿する文章 |
| プラットフォーム | マルチセレクト | X / Instagram / 両方 |
| 投稿予定日時 | 日付 | 時刻まで設定 |
| ステータス | ステータス | 草稿 / 確認待ち / 承認済み / 投稿済み |
| 添付画像メモ | テキスト | どの画像を使うか |
| 言語 | セレクト | 日本語 / 英語 / 日英混在 |

### ビュー

**① カレンダービュー「投稿カレンダー」**（メインビュー）
1. 「＋ ビューを追加」→「カレンダー」
2. 表示日付: 投稿予定日時
3. 名前: **「📅 投稿カレンダー」**
4. フィルター: ステータス ≠ 投稿済み

**② ギャラリービュー「草稿一覧」**
1. 「＋ ビューを追加」→「ギャラリー」
2. フィルター: ステータス = 草稿 または 確認待ち

**③ テーブルビュー「全投稿」**（デフォルト）

### 初期データの登録
`posts/` フォルダ内の草稿をここに転記してください（CEOが随時追加します）
